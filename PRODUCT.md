# Product

## Register

product

## Users

系统管理员和 API 网关运维人员，主要在桌面显示器上批量巡检渠道状态、模型配置、用量与运行日志，需要在高信息密度界面中快速发现和处理配置问题。

## Product Purpose

提供自托管的 Claude/OpenAI 兼容 API 网关及管理后台，集中管理渠道、模型映射、密钥、配额、计费和运行状态。成功意味着管理员能快速理解当前配置与健康状况，并以低出错率完成维护操作。

## Brand Personality

克制、专业、可靠。界面应让工具退居幕后，以清晰层级和一致组件帮助用户专注任务。

## Anti-references

避免高饱和彩色标签、过度圆润的消费级视觉、大块留白导致的信息密度下降，以及缺乏功能目的的装饰。

## Design Principles

- 高密度但不拥挤，优先支持桌面端快速扫读。
- 状态和数据层级清楚，不依赖颜色单独传递含义。
- 复用一致、熟悉的后台组件，不为视觉新奇重造交互。
- 关键操作和异常反馈明确，降低配置错误风险。
- 克制使用强调色，将注意力留给状态和主要操作。

## Accessibility & Inclusion

以 WCAG AA 为目标，保证文字对比度、键盘操作和清晰焦点状态；状态信息同时使用文字或形状表达，不只依赖颜色。
